本系列的主线一直围绕三大 Coding Agent——Claude Code、Cursor、Codex。把这三个吃透，足以应对绝大多数开发场景。但 AI 编程的生态远不止它们，外面还有一批各有特色的工具：有老牌的 IDE 插件，有另一款 AI 原生编辑器，还有一类「在浏览器里描述一句就生成可运行应用」的在线平台，以及把设计稿、截图直接转成代码的工具。

这一篇是收尾的生态扫盲，带你快速认识这些工具，知道它们是什么、适合谁，以及怎么和你已经学会的三大 Agent 搭配。不深入展开——目的是让你对整个生态有个全貌，遇到时不陌生、选用时有判断。

## **1. GitHub Copilot**

Copilot 是这波 AI 编程浪潮里最老牌的工具，由 GitHub 推出，最早以代码补全成名。你在 VS Code、JetBrains 等主流 IDE 里装上它的插件，写代码时它就在光标处给出灰色的补全建议，按 Tab 接受。

如今它早已不只是补全——也有了对话（Chat，问代码、让它解释或改）和能跨文件自主干活的 Agent 模式，能力向 Cursor、Codex 这类靠拢。它的优势在于和 GitHub 生态的深度绑定、企业接受度高、作为插件不必换编辑器。如果你的团队重度用 GitHub、又不想离开现有的 IDE，Copilot 是个稳妥的选择。对本系列的读者来说，把它理解成「以补全起家、装在你现有 IDE 里的 AI 助手」即可，它的 Agent 能力和三大主角思路相通。

## **2. Windsurf**

Windsurf 是另一款 AI 原生编辑器，和 Cursor 是同一赛道的竞品——都是把 AI 深度集成进编辑器，而不是当插件挂上去。它的核心是一个叫 Cascade 的 Agent，能理解你的代码库、跨文件自主改动、跑命令。

用过 Cursor 再看 Windsurf，会有很多相通之处：都有 AI 补全、都有自主 Agent、都强调对代码库的理解。两者各有拥趸，体验上各有细节差异。对你来说，理解了 Cursor 的核心概念（四模式、Rules、@引用、Agent），切到 Windsurf 几乎是平移——这也再次印证本系列反复说的：核心概念相通，换工具成本很低。所以本系列以 Cursor 为 AI 原生编辑器的代表深讲，Windsurf 知道它是同类、按需自行尝试即可。

国内用户还会遇到一批本土工具，值得知道它们存在。字节的 Trae 也是一款 AI 原生编辑器，和 Cursor 同类；阿里的通义灵码、智谱的 CodeGeeX 则是 IDE 插件形态，和 Copilot 同类，提供补全和对话。它们的卖点通常是国内网络访问稳定、对中文支持好、有免费额度，对刚入门、还不想付费或被网络问题困扰的人是不错的起步选择。能力和体验各有差距，但核心玩法仍是补全、对话、Agent 那一套——上手任何一个，本系列讲的概念都用得上。

## **3. 在线 AI 编程平台**

这是和三大 Agent 很不一样的一类，值得专门说说，因为它对零基础的人特别友好。v0、bolt.new、Lovable、Replit 等在线平台的玩法是：你不用装任何东西，打开浏览器，用一句自然语言描述你想要的应用或页面，它就在云端直接生成一个能运行、能预览的成品，还能一键部署上线。

它们和三大 Agent 的根本区别在「门槛」和「场景」。三大 Agent 是在你本地的真实项目里干活，适合严肃开发；这些在线平台是「从零、在浏览器里、快速做出一个能跑的东西」，适合做原型、验证想法、搭个小工具或落地页。你描述「做一个带深色模式的个人作品集页面」，几十秒后就能看到一个真实可点的页面，不满意接着用自然语言调。对完全不懂代码、只想快速把一个想法变成能看能用的东西的人，这类平台的即时成就感无可替代。

它们之间也各有侧重：v0 偏前端界面和组件，擅长生成漂亮的 UI；bolt.new、Lovable 偏全栈，能生成带后端、能交互的完整小应用；Replit 则是一个连开发、运行、部署都在浏览器里完成的在线环境，还自带 AI。不必都试个遍，挑一个顺手的做原型即可——它们的共同点是把「想法到能跑的成品」之间的距离压到了最短。

要清楚它们的边界：在线平台胜在快和零门槛，但生成的项目要做严肃的、长期维护的开发，往往还是要拉到本地、用三大 Agent 继续。所以更聪明的用法不是二选一，而是接力——下一节细说。

## **4. AI 辅助设计与截图转代码**

还有一类工具专攻「从视觉到代码」：你给一张设计稿、一张界面截图，甚至随手画的草图，它帮你转成对应的前端代码。Figma 等设计工具有相关的转代码能力，一些 AI 工具也支持你把截图丢进去、让它照着实现页面。

这对前端开发和不擅长从零写样式的人很有用：看到一个喜欢的页面、截个图，让 AI 照着做一个类似的；或者设计师出了图，直接转成代码起步。值得一提的是，三大 Agent 本身也越来越擅长这件事——你可以把设计截图直接发给 Cursor 或 Claude Code，让它照着实现。所以「截图转代码」与其说是一类独立工具，不如说是一种越来越普及的能力，在多个工具里都能用上。

## **5. 和三大 Agent 怎么搭配**

认识了这些工具，关键不是纠结选哪个，而是知道怎么让它们各展所长、接力配合。

一个很顺的组合是：用在线平台快速出原型——有了想法，先在 v0 或 bolt.new 里一句话生成一个能跑的初版，验证方向、看看效果；满意后把代码拉到本地，用 Cursor 或 Claude Code 继续做严肃开发，补功能、接数据库、做工程化；过程中需要照着某个设计实现页面，就把截图发给 Agent 让它实现。Copilot、Windsurf 则是「你已经习惯某个编辑器/生态」时的本地选择，和 Cursor 同类，按偏好挑一个即可。

这套接力的逻辑是：**在线平台管「快速从 0 到 1 出原型」，三大 Agent 管「在真实项目里做深做透」，设计转代码是贯穿其中的一种能力。** 想清楚每个工具最擅长的环节，按环节用对工具，比执着于「哪个工具最强」有意义得多。

## **6. 不要陷入工具焦虑**

最后说一句心态。AI 编程工具层出不穷，几乎每个月都有新工具、新功能冒出来，很容易让人陷入「是不是该换个更强的工具」的焦虑。

但工具再多，核心能力是相通的——补全、对话、自主 Agent、项目约定、上下文管理、MCP，这些概念在哪个工具里都一样。你把本系列三大 Agent 的核心吃透，再上手任何新工具几乎都是平移。所以别在工具间反复横跳、什么都浅尝辄止，选一两个主力工具用深用透，比追逐每个新工具有价值得多。新工具值得关注，但你的功力长在对这些通用概念的掌握上，而不是装了多少个工具。

## **7. 小结**

三大 Coding Agent 之外，AI 编程生态里还有：以补全起家、装在你现有 IDE 里的 GitHub Copilot；和 Cursor 同赛道的 AI 原生编辑器 Windsurf；让你在浏览器里一句话生成可运行应用、零门槛重原型的 v0、bolt.new、Lovable 等在线平台；以及把设计稿、截图转成代码的能力（它也已融入三大 Agent）。

认识它们的意义，不在于多装几个工具，而在于看清整个生态的分工：在线平台快速出原型，三大 Agent 在真实项目里做深做透，设计转代码贯穿其中——按环节让工具接力配合。守住这个判断，再加上对通用核心概念的扎实掌握，你就能不被层出不穷的新工具牵着走，而是稳稳地用对、用好。
