如果你已经习惯了 VS Code，那么上手 Cursor 几乎没有门槛——因为它本身就是基于 VS Code 改造的。但它又不只是装了个 AI 插件的 VS Code：补全、对话、自动改代码这些能力被深度织进了编辑器本身，用起来是另一种体验。很多人装上后只会用它的对话框，把一身本事浪费了大半。

这一篇带你把 Cursor 的核心用法一次打通：怎么从 VS Code 平滑迁移过来、Tab 补全为什么比传统补全快一个量级、三大交互入口分别在什么场景用、四种工作模式怎么回事，以及上手的第一个任务该怎么做。

## **1. Cursor 是什么**

Cursor 是一个 AI 原生的代码编辑器，由 Anysphere 公司基于开源的 VS Code 打造。所谓 AI 原生，意思是它不是在编辑器上挂一个 AI 插件，而是把 AI 的补全、对话、自主改代码当成核心功能重新设计了整个编辑体验。

因为底子是 VS Code，你在 VS Code 里的扩展、主题、快捷键几乎都能直接搬过来，界面也几乎一样，迁移成本极低。但它在这个底子上加了几样 VS Code 没有的核心能力：一个预测你下一步编辑的 Tab 补全、一个能读懂整个代码库并自主多文件改动的 Agent、以及随时唤起的行内编辑和对话。理解这一点，你就明白该把它当成一个更强的编辑器来用，而不是又一个聊天机器人。

## **2. 安装与从 VS Code 迁移**

去 Cursor 官网（`cursor.com`）下载对应系统的安装包：macOS 需要 12 及以上，Windows 需要 10 及以上，Linux 提供 deb 包。下载安装和装普通软件一样。

第一次打开 Cursor，它会主动问你要不要从 VS Code 导入——一键就能把你 VS Code 里的扩展、设置、快捷键、主题全部搬过来，搬完界面和你熟悉的 VS Code 几乎没差别。如果你是 VS Code 用户，这一步直接确认导入，省去重新配置的功夫。

## **3. Tab 智能补全**

Tab 补全是 Cursor 最先让你「哇」一下的能力。传统编辑器的补全只能补你正在打的这个词、这一行；Cursor 用的是它自研的专用补全模型，速度极快，而且预测的不是下一个词，而是**你下一步要做的整个编辑动作**。

具体体验是这样的：你改了一行代码，Cursor 会预判你接下来很可能要在别处做的相关改动（比如你改了一个变量名，它知道你下面几处引用也要跟着改），用灰色把建议显示出来，你按 `Tab` 就接受。更妙的是它能连续预测——接受一处后，它直接把光标跳到下一处该改的地方，你再按 `Tab`，就这样一路 `Tab` 下去，把一串相关改动连续做完，几乎不用手动移动光标。

刚上手时最该练的就是养成按 `Tab` 的直觉——很多人盯着补全建议却忘了它能连续接受，白白浪费了这个最高频的提效点。改代码时多留意那些灰色建议，对了就 `Tab`，你会发现大量机械的连带改动它都替你预判好了。

它的预判范围比你想的广：不只补你光标后面的内容，还能在一行中间插入、能根据你刚做的改动推断你接下来要在好几个地方做的一致性修改（重命名的连带引用、加了一个参数后各处调用的同步、改了类型后相关代码的适配）。这类机械但繁琐的连带改动，正是 Tab 最省事的地方——你只管确认方向对不对，按 `Tab` 让它一处处替你落实。

## **4. 三大交互入口**

除了 Tab，Cursor 让 AI 帮你干活有三个主要入口，分别对应三种粒度的需求，记住它们各自的快捷键和场景，就掌握了 Cursor 的主干。

`Tab` 是最轻的，补全你正在写的代码，前面讲过。`Cmd+K`（Windows 是 `Ctrl+K`）是行内编辑，选中一段代码按下它，用自然语言说要怎么改（「把这个函数改成异步的」「给这段加上错误处理」），它就地把这段改掉，适合对一小块代码做精准修改。`Cmd+I`（或 `Cmd+L`）唤起对话和 Agent，这是最重的入口——你可以和它聊代码、问问题，也可以让 Agent 接一个完整任务，它会自己读代码库、规划、跨多个文件改动、跑命令、迭代。

一句话区分：**补全用 `Tab`，改一小块用 `Cmd+K`，交一个完整任务用 `Cmd+I` 唤起 Agent。** 从轻到重，按你这次要干的活的大小来选。

`Cmd+K` 还有个容易被忽略的用法：在底部终端里按它，可以用自然语言生成 shell 命令——你不记得某条命令怎么写，直接说「把当前分支推到远程并设置上游」，它生成出来让你确认执行。无论是行内改代码还是终端生成命令，`Cmd+K` 的产物都会先给你看、由你确认，主动权在你手里。

## **5. 四种工作模式总览**

当你用 `Cmd+I` 唤起对话后，里面还分几种工作模式，决定 AI 是只聊、只读规划、还是放手动手。Cursor 目前主要有四种模式：**Ask** 只对话不改代码，适合你想先问清楚、理解代码、讨论方案；**Agent** 是默认模式，全自主——读代码库、规划、跨文件改动、跑命令、迭代，一个完整任务交给它；**Plan** 让它先制定一份可编辑的计划，你确认后再执行，适合复杂任务先对方向；**Edit**（行内的 `Cmd+K` 即属于这一类）做受控的局部修改。

在 Agent 输入框里按 `Shift+Tab` 可以在这些模式间切换。这套模式设计和 Claude Code 的「先 Plan 再执行、按需放手」思路是相通的——简单问答用 Ask，复杂任务先 Plan 再 Agent，局部精修用 Edit。这四种模式各自的深入用法和推荐工作流，下一篇会专门展开，这里先建立一个总览印象。

## **6. 第一个上手任务**

装好、迁移好，最好的熟悉方式是真的干一件小事。

先用 `Cmd+I` 唤起对话，让它带你认识这个项目：打开一个你手头的代码库，问「帮我解释一下这个项目的结构和主要功能」。Agent 会去读代码、给你一份概览——这是你接手任何陌生项目的第一步，也让你直观感受它读代码库的能力。

## **7. 界面与基础工作流**

Cursor 的界面布局和 VS Code 一致：左边文件树、中间编辑区、底部终端，右边多了一个 AI 对话面板。日常的基础工作流就是把前面几样配合起来用：写代码时靠 `Tab` 连续补全，改一小段时选中按 `Cmd+K`，做一个完整功能时 `Cmd+I` 交给 Agent（复杂的先切 Plan 模式对方向）。

有个高频且实用的细节：在对话里用 `@` 可以引用上下文。打一个 `@` 会弹出菜单，`@文件名` 引用某个文件、`@文件夹` 引用某个目录、`@Code` 引用某段符号、`@Web` 让它联网查、`@Docs` 引用某个库的官方文档、`@Git` 引用某次提交或改动。让 AI 基于你指定的具体代码和资料来回答，而不是凭空猜——这是让它答得准的最关键习惯。这套 `@` 引用和 Claude Code 是一个道理，上下文与记忆那一篇还会细讲。

还有一点关系到你对 AI 改动的掌控：Agent 或对话改完代码后，改动是以 diff 的形式呈现的，你可以逐处接受或拒绝，不满意整体回退。配合这点，你完全可以放手让它先改，再像审稿一样一处处过——这种「它先动手、你来验收」的节奏，是和 Cursor 协作的基本姿势。养成改完先看 diff 的习惯，比让它一路自动改下去更稳。

## **8. 常见问题**

**Q：我是纯 VS Code 用户，迁移会丢东西吗？**
基本不会。首次启动一键导入，扩展、设置、快捷键、主题都搬过来，界面几乎一样，照常用即可，多出来的是 AI 能力。

**Q：Tab 补全和普通补全有什么不一样？**
普通补全补当前词，Cursor 的 Tab 预测你下一步整个编辑动作，还能连续 `Tab` 跳到下一处该改的地方，把一串相关改动连着做完。

**Q：Cmd+K 和 Cmd+I 到底怎么选？**
改眼前一小块代码、要精准受控，用 `Cmd+K`；交一个要跨文件、要规划的完整任务，用 `Cmd+I` 唤起 Agent。

**Q：Agent 会不会乱改我的代码？**
它的改动你都能看到 diff、可以拒绝或回退。复杂任务先用 Plan 模式让它出方案、你确认方向后再执行，能大幅降低跑偏的概率。

## **9. 小结**

Cursor 的上手逻辑很清晰：它是一个你熟悉的 VS Code，加上了深度集成的 AI。迁移几乎零成本，关键是把它真正当编辑器用，而不是只盯着对话框——`Tab` 连续补全省掉大量机械改动，`Cmd+K` 精修局部，`Cmd+I` 把完整任务交给 Agent，复杂的先 Plan 后做。

把这三大入口和四种模式的对应关系记牢，按每次要干的活的大小选对工具，你就握住了 Cursor 的主干。后面几篇会把四种模式、Rules、上下文与记忆、MCP、云端代理这些逐个讲透，让你从「会用」走到「用得好」。
