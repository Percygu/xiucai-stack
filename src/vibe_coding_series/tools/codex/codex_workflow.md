Codex 的各项能力你已经一一过了一遍：多形态、AGENTS.md、命令、Skills、MCP、云端任务、沙箱审批。和前面 Claude Code、Cursor 一样，会用单个功能和能把它们串成顺手的工作流，是两码事。这一篇收尾，把 Codex 的能力收拢成一套实战节奏，再盘点一批高效技巧。

它不引入新功能，而是教你怎么把已有的功能组织起来，用出乘数效应。

## **1. 多步工具链自主干活**

Codex 最核心的工作方式，是自主地把一个任务拆成多步、串成一条工具链跑下来：读相关代码、理清思路、跨多个文件改动、跑测试验证、遇到失败再回头修，一直迭代到把活干成。你给的是一个目标，它走的是一整条链路。

这意味着你交代任务时，不必把每一步都拆好喂给它——说清楚要达到什么结果，让它自己规划这条链怎么走。当然，任务越复杂，一开始说得越清楚越好；模糊的目标会让它在自主链路上越走越偏。这也是为什么前面反复强调用 AGENTS.md 立规矩、复杂任务先讲清需求——它们都是在给这条自主工具链定好方向。

## **2. self-check 自检**

放手让 Codex 自主干，质量怎么保证？一个关键习惯是让它自检。它的工具链本就包含跑测试这一环，你可以进一步要求它在交付前对自己的产出做一遍检查：跑通测试、自查有没有遗漏边界情况、对照需求确认每条都做到了。

实操上，把自检写进你的要求里就行——比如交代任务时加一句「改完跑一遍测试确认通过，再自查一遍有没有漏掉错误处理」，或者干脆把这套自检流程沉淀进 AGENTS.md / 一个 Skill，让它每次都自动执行。让 AI 自己先把一遍质量关，再把结果交给你审，比它一股脑改完就甩给你要靠谱得多。这其实是「让 AI 写出的问题让 AI 自己先发现」的思路。

## **3. 跨形态接力**

Codex 多形态的真正威力，在于它们能接力——因为共享同一个账号和上下文，一个任务可以在不同形态之间无缝流转。

常见的接力姿势：在 ChatGPT 网页里委托一个云端任务，让它在沙箱里把初版做出来、开个 PR；回到本地用 CLI 把这个分支拉下来，边看边调那些需要你拿捏的细节；调好后在 GitHub 上用 `@codex review` 让它再审一遍。或者反过来——在 IDE 里写着写着遇到一个大活，甩到云端去并行跑，自己继续手头的事。不是四个割裂的工具各用各的，而是同一个 Codex 在最合适的形态间流转。

## **4. 一套完整工作流**

把前面的能力串起来，一个 Codex 任务从头到尾的节奏大致是这样。

开工前，先用 AGENTS.md 把项目规矩立好（新项目用 `/init` 生成起步版），让 Codex 一上来就懂你的项目。接一个任务时，按它的风险和难度设好挡位——日常用 `workspace-write` 沙箱 + `on-request` 审批，难题把推理强度调高，常用组合存成 Profile 一键切。复杂任务先把需求讲清楚、让它规划，简单的直接交。执行时它自主跑多步工具链，关键节点按审批策略问你。要求它交付前 self-check：跑测试、自查。能放手又想并行的活，丢云端委托，多个任务各自出 PR。最后用 `/review` 或 GitHub 上的 `@codex review` 把关，再合并。

这套节奏的内核和 Claude Code、Cursor 完全一致：人负责想清楚、做决策、把关，AI 负责执行、查找、跑链路；小步快跑、及时验证、用沙箱和审批兜底。三大工具殊途同归。

>
> **生图提示词（可直接发给 ChatGPT / 文生图工具）：**

## **5. 高效技巧盘点**

几条能立刻见效的习惯，不依赖某个具体功能。

**用 AGENTS.md 沉淀项目知识。** 把技术栈、结构、规范、常见任务的标准做法写进去，Codex 越用越懂你的项目，给的方案越贴合，这是回报最高的一次性投入。反复犯的某类错，补一条约束进去，让它边用边长。

**把常用配置存成 Profile。** 日常、只读分析、放手自动化几套沙箱+审批组合存好，`--profile` 一键切，省去反复改配置。

**按难易匹配算力。** 简单活用低推理强度省钱省时，难题调高让它深想；先用低档跑、卡住再升，大部分活低成本解决。

**把可复用流程做成 Skill。** 审查、发版、补测试这类反复用的流程，封装成 Skill，需要时自动或手动调用，还能跨工具、跨团队共享。

**能放手的丢云端、要互动的留本地。** 边界明确、可并行的活交云端委托，自己腾出手；要边看边调的留 CL/IDE。

**让它自检，但你做最终裁决。** 要求它跑测试、自查后再交付，但它的产出你仍要审——AI 是让你更高效的工具，最终质量靠你把关。

## **6. 与其他工具配合**

最后一句跨工具配合。Codex 和 Claude Code、Cursor 不是非此即彼——它们可以搭着用，而且共享越来越多的标准：`AGENTS.md` 是跨工具的项目约定，写一份多个工具都认；Skills 是跨工具的开放标准，一份 SKILL.md Codex、Claude Code 都能用；MCP 更是大家共通的连接协议。

所以实战里，你完全可以让它们各展所长：用 Codex 的云端委托并行跑大批任务、用它的 `@codex review` 审 PR，同时在本地用 Cursor 的编辑器体验写代码、用 Claude Code 处理终端里的深度自主任务，而它们都遵循你那一份 `AGENTS.md`、共用你那套 Skills。工具是手段，按手头的活挑最顺手的、让它们共享上下文协同，才是这一代 AI 编程工具的正确用法。

## **7. 小结**

Codex 的高效，不在某个单一功能，而在把多形态、AGENTS.md、Skills、沙箱审批、云端任务按节奏串起来：用 AGENTS.md 立好规矩，按任务设好沙箱审批挡位（存成 Profile 一键切），让它自主跑多步工具链、交付前 self-check，能放手的丢云端并行、要互动的留本地，最后用 review 把关——这套节奏跑顺，你和 Codex 的协作就从一问一答升级成了分工明确的高效配合。

而它和 Claude Code、Cursor 共享的那些标准（AGENTS.md、Skills、MCP），让你不必把自己绑死在一个工具上。守住「人做决策把关、AI 跑执行链路」这条分工，按活挑工具、让它们协同，你就真正驾驭了这一代 AI 编程工具。
