前两个项目，一个是静态主页，一个是存在浏览器本地的待办应用。这一篇我们做一个真正能上线、能给所有人访问的项目：个人博客系统。它用 Next.js 这个主流框架做，用 MDX 写文章，最后一键部署到云上，得到一个公开网址。

这个项目会综合用上前面学的工程方法——尤其是规划驱动开发。它比前两个复杂，但只要按规划走、分步推进，零基础也能做出来并成功上线。下面给出真实可运行的关键代码和完整流程，跟着做即可。

## **1. 规划先行**

复杂项目尤其要先规划。按工程篇的方法，先把博客想清楚：它要做什么——展示我写的文章，有一个文章列表页（首页）和文章详情页；文章用 Markdown 写（写起来最舒服）；要能方便地加新文章；最后要能部署上线、有公开网址。

把这个需求理清楚，就有了和 AI 讨论选型的基础。规划驱动开发的核心就是：先有清晰的蓝图，再动手，让 AI 照着蓝图建，而不是边想边乱建。

## **2. AI 辅助技术选型**

让 AI 帮你把技术方案定下来。

**Prompt：**
```
我要做一个个人博客，需求：文章列表页 + 文章详情页，文章用 Markdown 写，
方便加新文章，最后要能免费部署上线。我是新手，想用主流、好上手、社区资料多的方案。
请帮我做技术选型并说明理由：用什么框架、文章怎么组织、用什么部署。
```

AI 大概率会推荐：用 Next.js（最主流的 React 全栈框架，适合做这种带页面路由的网站），文章用 MDX（Markdown 的增强版，能在 Markdown 里用组件），部署到 Vercel（Next.js 官方平台，一键免费部署）。这套组合成熟、资料多、对新手友好，我们就照它来。选型时多问一句「为什么不用别的」，顺便理解取舍。

## **3. 搭起项目骨架**

让 AI 把 Next.js 项目搭起来。

**Prompt：**
```
帮我用 Next.js 搭一个博客项目（用 App Router）。先建好项目结构：
1. 文章用 MDX 文件存在 content/posts/ 目录下，每篇带标题、日期、摘要的 frontmatter
2. 首页展示所有文章的列表（标题、日期、摘要），点击进入文章详情
3. 文章详情页根据文章渲染正文
先把项目跑起来，放一两篇示例文章。
```

AI 会帮你创建项目、装依赖、写好基础代码。按它的指引 `npm install`、`npm run dev`，在浏览器打开 `localhost:3000`，就能看到博客首页了。

## **4. 用 MDX 写文章**

这个博客的妙处是：加一篇新文章，只要在 `content/posts/` 下放一个 `.mdx` 文件就行，完全不用动代码。一篇文章长这样：

```mdx
---
title: 我的第一篇博客
date: 2026-06-21
excerpt: 用 Vibe Coding 做了个博客，记录一下过程。
---

## 开篇

这是我用 AI 编程做出来的第一个能上线的项目……

正文用 **Markdown** 语法写，加粗、列表、代码块都支持。
```

上面 `---` 之间的是 frontmatter（文章的元信息：标题、日期、摘要），下面是用 Markdown 写的正文。首页会读取所有文章的 frontmatter 来生成列表，详情页则渲染正文。以后你写新文章，复制这个格式、改改内容即可——发布博客就变成了「写个 Markdown 文件」这么简单。

## **5. 列表页与文章详情**

博客的两个核心页面，背后是 Next.js 的两个关键能力。首页读取所有文章生成列表，关键代码思路是：

```jsx
// 首页：读取 content/posts 下所有文章，按日期排序，渲染成列表
const posts = getAllPosts();  // 读取所有 mdx 的 frontmatter
return (
  <ul>
    {posts.map(post => (
      <li key={post.slug}>
        <a href={`/posts/${post.slug}`}>
          <h2>{post.title}</h2>
          <time>{post.date}</time>
          <p>{post.excerpt}</p>
        </a>
      </li>
    ))}
  </ul>
);
```

文章详情页用的是动态路由——一个 `posts/[slug]` 的页面模板，能匹配任意文章。你访问 `/posts/我的第一篇博客`，它就找到对应的 mdx 文件、渲染成页面。这个「一个模板匹配所有文章」的机制，是 Next.js 做这类内容网站的核心，AI 会帮你写好，你理解它的作用即可。

## **6. 美化与完善**

骨架和内容都有了，再让 AI 把样式和体验做好。和个人主页那篇一样，把视觉要求说具体：

**Prompt：**
```
帮博客美化样式：
1. 首页文章列表用卡片式布局，干净现代，主色深蓝
2. 文章详情页排版舒适：合适的行高、字号、最大宽度，代码块有高亮
3. 加一个顶部导航和页脚
4. 适配手机
保持整体简洁、阅读体验好。
```

还可以继续加：文章标签分类、上一篇/下一篇、阅读时间估算、深色模式——每个都是一句 Prompt 的事。按「说清要什么、看效果、再调」的节奏推进，博客会越来越完善。

## **7. 部署上线**

这是这个项目最有成就感的一步——让全世界都能访问你的博客。Next.js 部署到 Vercel 几乎是零配置。

让 AI 一步步带你做：

**Prompt：**
```
我的 Next.js 博客做好了，想免费部署到 Vercel，请一步步教我：
从把代码推到 GitHub，到在 Vercel 导入项目、部署，到拿到公开网址。
我是新手，每一步都讲清楚。
```

它会带你走完：把代码推到 GitHub 仓库 → 在 Vercel 用 GitHub 登录、导入这个仓库 → Vercel 自动构建部署 → 几分钟后给你一个公开网址（类似 `你的博客.vercel.app`）。从此你这个博客全世界都能访问，而且以后你往 GitHub 推新文章，Vercel 会自动重新部署、更新线上内容。

## **8. 复盘**

这个项目把规划驱动开发完整走了一遍，也第一次体验了「部署上线」的全流程。

它和前两个项目最大的不同，是**复杂度上来后，规划的价值凸显了**。如果不先理清需求和方案就让 AI 乱建，很容易建出一团乱麻；而先想清楚「列表页+详情页+MDX 文章+部署」这条主线，再让 AI 一步步照着搭，过程就清晰可控。这印证了工程篇的核心：AI 写得越快，越要靠规划把住方向。

另一个收获是体会了全栈和部署。博客虽然内容是静态的，但它有了真正的页面路由、文件驱动的内容管理、以及上线流程——这些是做更复杂 Web 应用的基础。下一个全栈项目，会在这个基础上再加真正的后端和数据库。

## **9. 小结**

个人博客系统带你完成了三个第一次：第一次用 Next.js 这样的主流全栈框架、第一次体验规划驱动开发在复杂项目里的价值、第一次把作品真正部署上线让所有人能访问。你从理清需求、AI 辅助选型起步，搭起项目骨架、用 MDX 把发文章变成「写个 Markdown 文件」、用动态路由做出文章详情，最后一键部署到 Vercel 拿到公开网址。

最关键的体会是：**项目越复杂，规划越重要。** 先有清晰的蓝图，再让 AI 照着分步搭建，复杂项目也能稳稳落地。带着这套规划驱动的能力，下一篇我们挑战这个系列最综合的项目——一个带用户登录、数据库的全栈 Web 应用。
